React Component

An oftern pure, mostly declarative function that can take in props as input and returns JSX as output

  • Pure Function: Does not have side effects, always gives the same output for the same input
  • Impure Function: Has side effects, does not always give the same output for the same input

When you need a pure function to have a side effect, use a:

React Effect

A side effect of rendering that needs to run outside of the pure rendering function.

To attach a side effect to a pure function:

UI: React UseEffect Hook: takes two parameters:

useEffect(() => { //do something }, DEPENDENCY_ARRAY)

  • first parameter: "//do something" is what is ran after rendering is complete
  • second parameter: DEPENDENCY_ARRAY; The effect will run when the dependency array changes.
UseEffect Hook will only run if the dependendy array changes. If it does not change, the UseEffect Hook will not run again.

Three Options for the dependency array:

  • No dependency array

    useEffect(() => { //do something })

    This will run the effect after every render.

    Run on every render (we don't usually do this)
  • Full dependency array

    useEffect(() => { //do something }, [DEPENDENCY1, DEPENDENCY2])

    This will run the effect only when the dependencies change.

    Run on first render, and later renders where some or thing has changed since the last render
  • Empty dependency array

    useEffect(() => { //do something }, [])

    This will run the effect only once, after the initial render.

    Run on first render

If Strict Mode is being used: (should leave strict mode on) this happens when strict mode is on:

Does some extra checks, only in development mode, to help catch bugs
One of those checks is to load, unload, and load again when a component first loads in.

When do we use an Effect?

For behavior triggered by rendering that isn't part of rendering, usually to sync with an external system

Most common examples:

  • 1. Fetching data from the backend
  • 2. Starting a loading-in animation
  • 3. To interface with a DOM-manipulation-based system or widget

Warning:

Before you reach for useEffect, make sure you couldn't just do it in an event listener or rendering code!

  • Are you sure it's triggered by rendering?

    If not, then it's probably triggered by a user event, so put it in the event handler instead.

  • Are you sure it's not part of rendering?

    If it is part of rendering, then put it directly in the rendering function instead.
